Things to know:

  • Labels in <form> get display: block;
  • Inputs in <form> get display: block; and width: 100%;
  • Use <span class="required-asterisk">*</span> in labels for required inputs.
  • You can use .input-list to wrap a single checkbox/radio (when you need margin). It's currently used in this page ("Do you agree?" checkbox).

Input Groups

To use our forms styles you must understand input groups. These are self-contained groups which visually collect related core elements together. Input groups are powerful ways to re-order content based on your UI needs without needing to create new custom names. At a minimum, an input group is comprised of two elements: typically a label and an input. Here's an example of that:

{% highlight html linenos %}
{% endhighlight %}

Input groups utilize flexbox to work its magic. A standard .input-group element will stack child elements in a column fashion. This is because, by default, our approach with forms are to stack labels, inputs, description text, etc on top of each other, along the left edge.

Unfortunately unique layouts arise and the standard layout above won't solve all of our problems. While we can't foresee every possible layout, we can create a number of ready-made tools for people to use so we don't have to resort to creating custom classes. The biggest tool at our disposal is a flexbox-based grid system.
