Buttons

Base Styles

Primary

Primary buttons are used to provide a clear calls-to-action to a primary task. These buttons help guide the user to suggested next steps.

Primary, ._primary
{% highlight html linenos %} {% endhighlight %}

Secondary

Secondary buttons are used to provide alternate, less-important actions to the users. These buttons are purposefully muted in tone and color in order to not grab the user's attention.

Secondary, ._secondary
{% highlight html linenos %} {% endhighlight %}

Confirmation

Confirmation buttons are used for the specific use cases of completing a purchase, publishing a new product, or updating an existing product.

Confirmation, ._confirm
{% highlight html linenos %} {% endhighlight %}

Danger

Danger buttons are used for the specific use cases of warning a user that a potential action they are about to take is irreversible, such as deleting a job listing, removing a team member, or deleting their account.

Danger, ._danger
Danger (Outlined), ._danger._outlined
Danger (Clear), ._danger._clear
{% highlight html linenos %} {% endhighlight %}

Outlined

Outlined buttons are a stylistic variant of the ._secondary button class. In some instances, we may want to use a even simpler button style to draw attention to the primary call-to-action. This style is mainly used within small UI actions that tertiary in importance and on marketing pages.

Outlined, ._outlined
{% highlight html linenos %} {% endhighlight %}

Clear

Clear buttons are used when we don't want to visually style an action or link as a button, yet we want to provide ample click area and visual white-space around the link to separate it from surrounding content. This is mainly used within small UI actions.

Clear, ._clear
{% highlight html linenos %} {% endhighlight %}
