Form Labels

Labels should be written in sentence case. For usability reasons, if a label is connected with an input, the for attribute should be filled in. The for attribute references the input's id value. Additionally if you would like to add required or optional labels, whenever possible use the full word label instead of asterisks as this allows it to be more flexible for translations.

Input Groups

Input groups are a key layout element for our forms styles. Many of the styles that you've seen above are only possible because of input groups. So what are input groups? They are self-contained groups of related input elements which are visually collected together. They are a versatile, powerful control to arrange content based on your UI needs without needing to create new custom names. At a minimum, an input group is typically comprised of at least two elements: a label and an input.

{% highlight html linenos %}
{% endhighlight %}

The power of input groups lie in their use of flexbox. All forms, by default, are stacked elements (i.e. not inline, in a row) along the left edge.

Button Placement

Default

The default button placement within a form is left-aligned, with the primary call-to-action (CTA) being listed first, followed by the secondary CTA, and so on. The .button-group only addresses spacing around the buttons. It does not add any visual decoration.
