    • Helpers
      Definition

      Helpers help you build components and views, they are not meant to be used on their own (except _utilities.less classes).

      If you're working on a specific .less file which need to use helpers, just import _less/helpers/_helpers.less, it contains all the helpers.

      Example

      Build a tooltip in /components/_tooltips.less using:

      • .triangle() to create the arrows
      • .ellipsis() to prevent text overflow
      • .animate-in() for a little fade in animation
      • visibility classes to hide the tooltip on mobile
      • _animations.less
        Basic animations used throughout SO
      • _grids.less
        A small regular fluid grid system that we (should) use on all the pages.
      • _helpers.less
        Loads all the helpers files, needed when you work on a file that does not import base.less.
      • _mixins.less
        .triangle(), .ellipsis(), .transition(), .clearfix()... built to avoid repetition.
      • _reset.less
        Reset base css, must be imported at the beginning of base.less
      • _utilities.less
        Floats, visibility, alignment, responsive, helper classes... These are the only classes that you can use directly in the html.
    • Views
      Definition

      Views are page specific less plus component overrides. The structure inside this folder should follow SO navigation. These files need to be loaded into base.less.
